# CDP 前端 UI 重构设计说明

## 概述

本设计说明用于定义 CDP 前端下一阶段的视觉升级方向，首期以普通工作台为核心，后续再延展到批量模式。整体目标气质如下：

- 年轻，但不轻浮
- 简约，但不冷淡
- 有大厂产品感，而不是传统企业后台感
- 轻柔、朦胧、有层次、会呼吸
- 禁止使用蓝色或紫色系作为主视觉强调色

本次选定的结构方向为：

- 左侧轻导航
- 中部主舞台
- 右侧上下文抽屉

这不是一次彻底推翻式重构，而是一次有节制的产品演进。目标是在保留现有工作效率的前提下，让界面更现代、更细腻、更有情绪质感。

## 目标

- 提升产品的第一眼观感和整体质感
- 建立更有辨识度、更有品牌感的界面印象
- 提升普通模式工作台的留白感和视觉层级
- 减少长期固定侧栏带来的压迫感
- 保持高频操作下的使用效率
- 沉淀一套后续可迁移到批量模式的视觉系统

## 非目标

- 不重写业务逻辑
- 不替换 Element Plus
- 不调整后端 API 设计
- 不把工具做成过度营销化、过度表演化的页面
- 不使用高饱和、强刺激、霓虹感的视觉表达

## 设计方向

整体体验应更接近一个安静、克制、有高级感的创作型工作台，而不是信息密集、层层堆叠的传统配置台。视觉语言应传达柔和、自信、精准的产品气质。

界面应重点传递以下感受：

- 用柔和暖意替代技术冷感
- 用现代产品精致度替代内部工具粗糙感
- 用清晰结构替代视觉堆叠和噪声

情绪参考是“带有年轻品牌语气的雾感高端生产力产品”。装饰应服务于氛围和层次，而不是抢夺内容主导权。

## 布局策略

### 顶部栏

顶部区域从当前偏漂浮的工具条，调整为一条更轻、更整合的产品栏。

建议承载内容包括：

- 产品名称或简洁的工作区标题
- 模式切换
- 后端健康状态
- 主要操作，如复制、导出、外链跳转等

这一区域应与页面整体表面融为一体，而不是像单独悬浮的后台工具条。

### 左侧轻导航

当前左侧组件面板需要明显减重。

新的结构建议是：

- 常驻一条窄型导航轨道，作为整体壳层的一部分
- 组件搜索与组件选择使用柔和的可展开面板承载

这样可以在不牺牲组件添加效率的前提下，让整个工作区更通透、更有呼吸感，避免左侧形成厚重且长期占位的工具栏印象。

### 中部主舞台

中部区域应成为整套产品最核心的视觉舞台。

它承载的内容包括：

- 当前配置画布
- 节点卡片
- 逻辑连接关系
- 空状态引导

这一区域需要获得更多横向与纵向留白。节点不应再像密集表单容器堆叠在后台布局里，而应更像漂浮在安静舞台上的模块化工作单元。

### 右侧上下文抽屉

当前常驻右侧大栏建议改为更具上下文语义的详情抽屉。

建议行为如下：

- 当没有聚焦对象时收起或弱化存在感
- 当选中节点或主动查看结果时展开
- 在摘要视图和 JSON 视图之间快速切换

这样可以保留结果查看和细节操作能力，同时不再长期压缩中部主舞台的展示空间。

## 视觉语言

### 配色系统

整体色盘应以浅色、低饱和、轻雾感为基础。

中性色建议包括：

- 奶雾白
- 暖云灰
- 麦燕麦米色
- 烟雾灰
- 温墨灰文字色

主强调色建议使用：

- 雾珊瑚
- 淡杏橘
- 柔桃砂

辅助支持色建议使用：

- 浅鼠尾草绿
- 亚麻灰金

配色规则如下：

- 禁止蓝色强调
- 禁止紫色强调
- 避免纯黑与纯白的硬对比
- 强调色只作为轻声提示，而不是高音量宣告

### 表面质感

界面表面建议采用以下处理方式：

- 低对比层叠
- 轻透半透明填充
- 合理范围内的细腻模糊
- 大而散的柔光阴影
- 很淡的边界定义

卡片和面板应有轻微悬浮感，但不能出现油亮、喧闹或廉价的质感。

### 装饰语言

允许使用的装饰元素包括：

- 稀疏点阵
- 极细渐变线
- 轻气氛渐变
- 偶发性的模糊光斑背景

这些元素更适合出现在页面外壳、舞台背景和空状态区域，不应铺到信息密集的表单后方，以免影响可读性。

## 字体策略

字体表现应更精致、更现代，但不能牺牲工具场景下的清晰度。

原则如下：

- 降低当前系统中过大、过戏剧化的标题比例
- 更多依靠留白、字号梯度和字重层级来建立秩序，而不是单靠大字
- 文字颜色使用柔和深色，而不是纯黑
- 标签要小、稳、安静
- 标题要优雅，但仍以效率为先

建议层级包括：

- 产品标题
- 区域标题
- 卡片标题
- 正文内容
- 微标签
- JSON 与表达式类的等宽输出

## 组件策略

### 节点卡片

节点卡片应成为整个界面的主视觉模块。

改造重点包括：

- 外壳更柔和
- 头部结构更干净
- 阴影更克制
- 内部间距更舒展
- 标题、状态、操作分组更清晰
- 收起态与展开态的区分更明确

节点卡片最终应呈现出“高级、清楚、易扫读”的工具气质，而不是为了好看而增加装饰负担。

### 连接关系与运算符

节点间逻辑关系需要更优雅、更精准地被表达出来。

改造建议包括：

- 用细渐变线替代厚重的视觉分隔
- 把运算选择做成更轻巧的分段控件或柔和胶囊
- 让连接区本身看起来更有设计感和空气感

### 搜索与表单输入

输入类组件应更安静、更细腻。

改造方向包括：

- 边框更轻
- 聚焦态使用雾珊瑚色柔光提示
- 圆角比例更协调
- 非激活态减少存在感

### 操作按钮

按钮的层级区分应更多依赖“信心等级”，而不是依赖颜色音量。

建议规则如下：

- 主按钮使用低饱和暖色填充
- 次按钮使用轻透表面与描边
- 危险操作保持可识别，但不过度刺眼
- hover 反馈应克制、精致、有产品感

### 摘要与 JSON 结果查看

结果查看默认应优先展示摘要，因为摘要更容易理解，也更适合日常使用。

JSON 继续保留，但视觉优先级下降为次级视图。

右侧抽屉应优先强调：

- 简洁的摘要卡片
- 清晰的标签结构
- 易读的逻辑表达式
- 方便的复制与导出操作

## 动效策略

动效应服务于柔和、稳定与高级感。

允许的动效包括：

- 卡片进入时轻微淡入上浮
- 右侧抽屉柔和滑出
- hover 时极轻的抬升反馈
- 状态切换时平顺过渡

应避免以下类型：

- 过于弹跳和玩具化的反馈
- 大幅缩放
- 夸张发光动画
- 持续存在、会分散注意力的装饰性动态

## 普通模式首期落地

首轮实现建议以普通模式为核心，因为它最能完整体现这套新视觉系统。

第一阶段建议范围包括：

- 重构顶部栏结构
- 重做左侧轻导航视觉
- 重做中部舞台外壳与节点卡片
- 重做右侧上下文抽屉样式与交互存在感
- 统一全局色彩令牌和基础组件样式

这一步应在不改变核心业务能力的前提下，形成用户可感知、可对比的明显升级效果。

## 批量模式后续延展

批量模式后续应继承同一套设计系统，包括：

- 相同的表面语言
- 相同的强调色规则
- 相同的排版体系
- 相同的动效语汇

它的空间结构可以继续保持管线式特点，但整体气质必须与普通模式保持同一家族感。

## 测试与验证

验证重点应覆盖：

- 常见笔记本尺寸下的桌面可读性
- 较小宽度下的布局稳定性
- 右侧抽屉工作流的可见性与可用性
- 节点选中态与未选中态的区分度
- 摘要视图与 JSON 视图的阅读体验
- 装饰元素在真实数据密度下是否仍然克制

## 当前落地状态

- 普通模式已完成首轮结构重构与视觉升级
- 批量模式已完成壳层与视觉语言对齐
- 后续如需继续推进，可再拆第二期交互动效与批量模式深度结构升级

## 风险与应对

风险：柔和视觉语言可能导致控件对比度不足。

应对：交互状态必须强于背景装饰，尤其是聚焦、选中、危险操作等关键状态。

风险：将常驻右栏改为抽屉后，可能影响熟练用户的查看效率。

应对：保留快速打开入口，并确保摘要与结果切换足够直接、足够快。

风险：雾感装饰如果使用过度，容易显得虚、散、廉价。

应对：把氛围控制在页面壳层和背景层，真正承载工作内容的表面必须保持足够清晰。

## 实施建议

建议以设计驱动的方式推进前端首轮改造，并以普通模式为中心展开。

推荐实施顺序如下：

1. 建立共享视觉令牌与页面壳层规则
2. 将普通模式结构调整为左侧轻导航、中部主舞台、右侧上下文抽屉
3. 重做节点卡片、连接关系与关键操作控件
4. 优化摘要区与 JSON 查看区的承载方式
5. 在普通模式验证通过后，把同套视觉系统推广到批量模式
