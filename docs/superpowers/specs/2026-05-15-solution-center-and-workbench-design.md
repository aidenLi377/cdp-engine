# 方案中心与工作台方案使用功能设计

## 1. 背景

当前系统已经具备可视化点选能力，用户可以通过拖拽节点、填写参数、设置交并差关系来生成圈人 JSON。

现阶段的主要问题是：

- 同一类业务套路需要重复搭建
- 用户已经沉淀出常用的组合结构，但系统无法复用
- 完整画布适合搭建，不适合高频快速出包

本设计的目标是在现有系统上新增一套“方案”能力，把常用圈人结构沉淀为可复用模板，同时保持工作台和方案配置的边界清晰。

## 2. 目标

本次设计要解决四件事：

1. 允许用户把一套完整画布结构沉淀为方案
2. 允许用户在单独入口中管理和配置方案
3. 允许用户在工作台直接使用正式方案快速出包
4. 让工作台使用态只暴露少量高频可编辑参数，而不是整套结构

## 3. 范围

### 3.1 本期范围

- 新增“方案中心”一级入口
- 支持方案草稿与正式方案两种状态
- 支持从工作台将当前画布保存为方案草稿
- 支持在方案中心编辑方案结构
- 支持在方案中心配置“工作台展示参数”
- 支持将草稿发布为正式方案
- 支持在工作台使用正式方案
- 支持在工作台恢复方案默认值
- 支持在方案中心预览工作台使用态

### 3.2 本期不做

- 用户体系
- 个人方案/团队方案权限区分
- 正式的分享机制
- 方案版本历史
- 方案分类体系
- 工作台内编辑方案定义

未来演进方向为：以“个人方案”为主，通过分享能力将方案分发给其他人。

## 4. 核心设计结论

### 4.1 一级入口结构

系统首页调整为三个一级入口：

- 可视化点选
- 矩阵装配
- 方案中心

### 4.2 角色分工

- 工作台负责“做事”
  - 自由搭建
  - 使用正式方案
- 方案中心负责“做模板”
  - 新建方案
  - 编辑方案
  - 管理草稿
  - 发布正式方案

### 4.3 方案与人群包名称关系

- 方案名与最终生成人群包名是两个概念
- 进入工作台方案使用态时，默认人群包名等于方案名
- 用户可以修改本次人群包名称
- 修改本次人群包名称不影响方案定义

## 5. 方案生命周期

### 5.1 状态定义

方案仅保留两种状态：

- 草稿方案
- 正式方案

### 5.2 方案来源

草稿方案可以从以下两条路径产生：

1. 在方案中心手动新建空白草稿
2. 在工作台中将当前画布“存为方案草稿”

### 5.3 正式方案产生规则

- 正式方案只能在方案中心发布
- 草稿方案编辑完成后，点击“发布正式方案”
- 发布成功后，方案进入工作台方案列表

### 5.4 编辑正式方案规则

- 正式方案不能直接修改定义
- 在方案中心编辑正式方案时，系统先生成一个“编辑草稿”
- 用户在编辑草稿上完成结构调整与展示参数配置
- 再次发布后，用当前草稿覆盖原正式方案

该规则的目的是避免正式方案在半编辑状态下影响工作台用户使用。

## 6. 页面结构设计

### 6.1 方案中心

方案中心为三栏布局：

- 左侧：方案列表区
- 中间：方案编辑区
- 右侧：方案配置区

#### 6.1.1 左侧：方案列表区

按状态分组展示：

- 正式方案
- 草稿方案

支持：

- 搜索
- 状态筛选

每个方案项展示：

- 方案名
- 状态
- 更新时间
- 来源

来源示例：

- 手动新建
- 来自工作台
- 来自正式方案编辑草稿

每个方案项的操作定义如下：

- 草稿方案
  - 编辑
  - 复制
  - 删除
  - 发布正式方案
- 正式方案
  - 查看
  - 生成编辑草稿
  - 复制
  - 删除

#### 6.1.2 中间：方案编辑区

该区域本质上复用现有可视化点选画布能力，但页面语义调整为“方案结构编辑器”。

支持能力：

- 添加节点
- 删除节点
- 调整节点顺序
- 配置交并差关系
- 展开/收起节点
- 填写节点参数
- 清空画布
- 撤销/重做

该区域用于定义方案结构本身，而非用于一次性出包。

#### 6.1.3 右侧：方案配置区

右侧拆分为三个部分：

1. 基本信息
2. 工作台展示参数设置
3. 保存与发布

基本信息包括：

- 方案名称
- 当前状态
- 来源说明
- 若为“正式方案编辑草稿”，需展示“发布后将覆盖原正式方案”的提示

工作台展示参数设置包括：

- 按节点分组展示可配置参数
- 以单个参数为粒度勾选
- 仅勾选项会出现在工作台方案使用态中

保存与发布包括：

- 保存草稿
- 发布正式方案

### 6.2 工作台

工作台保留现有“可视化点选”入口，但新增正式方案列表区。

#### 6.2.1 左侧结构

左侧分为两个区域：

- 正式方案列表
- 行为组件列表

两者必须有明确视觉区分，避免用户将“方案”误解为一种“节点组件”。

#### 6.2.2 工作台两种使用路径

工作台支持两条路径：

1. 自由搭建
2. 使用正式方案

#### 6.2.3 方案使用态

用户点击正式方案后，进入工作台“方案使用态”。

进入规则：

- 当前画布为空时，直接载入方案
- 当前画布非空时，先弹确认，再清空并载入方案

进入方案使用态后：

- 中间区不再以完整画布为主视图
- 中间区改为轻量表单页
- 表单按节点分组展示
- 每个节点下只显示方案中心勾选过的参数

右侧结果区继续保留：

- 参数摘要
- JSON 预览
- 复制结果
- 相关结果操作

#### 6.2.4 工作台允许的方案相关动作

工作台中与方案相关的能力仅包括：

- 载入正式方案
- 修改本次使用参数
- 修改本次人群包名称
- 恢复方案默认值
- 将当前自由画布存为方案草稿

工作台中不允许：

- 编辑方案结构定义
- 配置工作台展示参数
- 发布正式方案
- 覆盖正式方案定义

## 7. 模式1使用体验定义

虽然前期讨论中存在“模式1/模式2”的说法，但落地时不再在工作台中显式暴露“方案编辑模式”。

产品语义改为：

- 工作台：方案使用态
- 方案中心：方案编辑态

工作台中的方案使用态需要满足以下原则：

- 轻量
- 稳定
- 只暴露高频调整项

工作台使用态中，中间区展示内容包括：

- 当前方案名
- 本次人群包名输入框
- 按节点分组的可编辑参数
- 恢复方案默认值按钮

## 8. 工作台展示参数配置规则

### 8.1 配置方式

- 在方案中心中完成配置
- 按节点分组
- 按单个参数勾选

### 8.2 工作台显示规则

- 仅被勾选的参数，才会出现在工作台方案使用态中
- 没有被勾选的参数继续沿用方案默认值

### 8.3 联动规则

“工作台展示参数设置”必须与当前方案结构实时联动：

- 节点被删除后，对应参数组自动消失
- 节点类型变更后，不再存在的参数自动移除
- 发布前不允许保留无效展示项

可选提示文案：

- 因方案结构变更，已自动移除不可用的展示项

## 9. 方案默认值与恢复机制

### 9.1 默认值定义

方案在发布为正式方案时，当前保存的参数值即为该方案默认值。

### 9.2 工作台恢复默认值

工作台中的“恢复方案默认值”定义如下：

- 恢复当前方案保存时的默认参数值
- 恢复默认人群包名称
- 只影响本次使用态
- 不影响方案本身

## 10. 关键交互规则

### 10.1 需要确认弹窗的场景

- 当前画布非空时载入正式方案
- 方案中心切换到其他方案且当前草稿未保存
- 删除草稿方案
- 删除正式方案
- 发布正式方案
- 用编辑草稿覆盖原正式方案

### 10.2 使用轻提示的场景

- 保存草稿成功
- 存为方案草稿成功
- 恢复方案默认值成功
- 发布正式方案成功
- 自动移除不可用展示项

### 10.3 预览工作台使用态

方案中心增加“预览工作台使用态”能力，建议采用右侧抽屉或同级轻预览层，而非重阻断弹窗。

预览中展示：

- 工作台会看到的节点分组
- 每组下会展示的参数
- 默认人群包名称效果

## 11. 典型流程

### 11.1 从工作台沉淀方案

1. 用户在工作台自由搭建一套结构
2. 点击“存为方案草稿”
3. 系统提示“已保存到方案中心草稿”
4. 草稿进入方案中心草稿区
5. 用户后续在方案中心补充方案名、展示参数、发布

### 11.2 方案中心发布正式方案

1. 用户打开草稿方案
2. 编辑结构与默认参数
3. 勾选工作台展示参数
4. 预览工作台使用态
5. 点击“发布正式方案”
6. 发布成功后，方案进入工作台正式方案列表

### 11.3 工作台使用正式方案

1. 用户进入工作台
2. 点击左侧正式方案
3. 若工作台已有内容，则先确认是否替换
4. 进入方案使用态
5. 修改少量参数并生成本次人群包
6. 如需回退，点击“恢复方案默认值”

### 11.4 编辑正式方案

1. 用户在方案中心查看正式方案
2. 点击“生成编辑草稿”
3. 在草稿中完成结构调整与展示参数配置
4. 点击“发布正式方案”
5. 用当前草稿覆盖原正式方案

## 12. 第一版实现建议

第一版建议只实现以下必要能力：

- 新增“方案中心”一级入口
- 方案中心支持草稿与正式方案管理
- 工作台支持正式方案列表与方案使用态
- 工作台支持“存为方案草稿”
- 方案中心支持工作台展示参数配置
- 方案中心支持预览工作台使用态
- 正式方案编辑采用“生成编辑草稿”机制

第一版不建议同时引入以下增强项：

- 分享链路
- 多级分类
- 用户归属
- 历史版本
- 审批流

## 13. 未来扩展方向

后续可沿以下方向演进：

- 引入个人方案概念
- 支持方案分享给其他用户
- 方案复制后保留来源关系
- 方案分类与搜索增强
- 正式方案版本历史与回滚

## 14. 设计总结

本方案的核心原则是：

- 方案中心负责定义方案
- 工作台负责使用方案
- 草稿与正式方案严格分层
- 工作台只消费正式方案
- 使用态尽量轻，编辑态集中管理

该设计既保留了当前可视化点选的自由度，也为后续业务模板沉淀、个人方案、分享机制提供了清晰演进路径。
