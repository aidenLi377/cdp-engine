# CDP Gallery White 全站前端视觉重构设计

## 背景

当前前端是 Vue 3 + Element Plus 应用，主要界面包括登录页、全局应用壳、工作台、方案中心、任务中台，以及工作台内的批量模式、动态表单、抽屉和弹窗。

现有 `cdp-global.css` 末尾的 “Apple Clean Pro” 主题虽然希望营造苹果式质感，但将橙色同时用于页面背景、分隔线、阴影、卡片渐变、选中态和按钮。暖白 `#fffdf8`、棕色文字 `#211813`、橙色阴影及大面积橙色渐变叠加后，使界面整体偏黄、偏暖且显脏。

本次重构将全站统一为已经确认的 **Gallery White + A1 Signal Orange + P1 System Semantics**：冷白画布、黑色结构、中性灰层级，橙色只承担焦点与选中信号，绿/黄/红只承担状态语义。

## 目标

- 登录页、工作台、方案中心、任务中台和所有可达子界面使用同一套清新、简约、克制的视觉语言。
- 达到接近苹果官网的产品展示感：纯净底色、明确字号层级、充足留白、精确对齐和低噪声表面。
- 保留现有信息架构、交互流程、业务逻辑、接口和用户数据。
- 让橙色成为稀缺且明确的信号色，而不是环境色或装饰色。
- 通过主题令牌、组件规则和自动化测试阻止暖色污染回归。

## 非目标

- 不修改后端、API、认证、权限或数据存储。
- 不重做工作台、方案中心和任务中台的业务流程。
- 不新增暗色主题或主题切换能力。
- 不进行与本次视觉统一无关的组件重构。
- 不引入大体积动画库、外部字体服务或装饰性图片资源。

## 已确认的视觉方向

### Gallery White

页面使用纯白和冷灰建立空间，不使用暖白、米白、奶油色或棕色。结构由黑色标题、灰色正文、中性分隔线和留白完成；卡片不依赖彩色背景或强阴影制造层次。

### A1 Signal Orange

主要行动按钮保持黑色。橙色只用于：

- 键盘焦点环和输入焦点边界；
- 当前选中的节点、方案、任务或导航线索；
- 必须吸引注意的文字链接或小型指示器；
- 灰阶图表中的唯一重点序列。

橙色不得用于页面底色、侧栏底色、卡片底色、大面积选中底色、装饰渐变或阴影。

### P1 System Semantics

- 绿色仅表示成功、连接正常或执行完成。
- 黄色仅表示等待、人工确认或非阻断警告。
- 红色仅表示失败、离线、删除风险或阻断错误。
- 状态色优先使用图标、圆点、文字和窄边线，不染色整张卡片或大面积容器。
- 图表默认使用灰阶；除非多序列比较确实需要，否则不增加第二套品牌色。

## 设计令牌

主题令牌使用语义命名，组件不得继续直接引用旧的暖色十六进制值。

### 中性色

| 用途 | 令牌 | 值 |
| --- | --- | --- |
| 页面画布 | `--ui-canvas` | `#ffffff` |
| 次级分区 | `--ui-fill` | `#f5f5f7` |
| 表面 | `--ui-surface` | `#ffffff` |
| 主文字/主行动 | `--ui-ink` | `#1d1d1f` |
| 次级文字 | `--ui-text-secondary` | `#6e6e73` |
| 弱化文字 | `--ui-text-tertiary` | `#86868b` |
| 控件边界 | `--ui-control-border` | `#d2d2d7` |
| 分隔线 | `--ui-divider` | `#e8e8ed` |

### 强调与状态色

| 用途 | 令牌 | 值 |
| --- | --- | --- |
| 焦点/选中 | `--ui-accent` | `#ff6b35` |
| 焦点环 | `--ui-accent-ring` | `rgba(255, 107, 53, 0.10)` |
| 成功 | `--ui-success` | `#34c759` |
| 等待/警告 | `--ui-warning` | `#ffcc00` |
| 错误 | `--ui-danger` | `#ff3b30` |

### 形状、阴影和动效

- 普通控件圆角为 8px；卡片和面板为 10–12px；胶囊形只用于分段导航、状态标签和紧凑行动按钮。
- 普通卡片默认无阴影，仅使用 1px 中性边界。
- 浮层、抽屉和模态框允许使用中性阴影，例如 `0 18px 50px rgba(0, 0, 0, 0.10)`。
- 悬停层次只允许中性阴影，禁止橙色或其他彩色阴影。
- 交互动效为 180–260ms，采用 `cubic-bezier(0.16, 1, 0.3, 1)`；遵循 `prefers-reduced-motion`。

## 字体与信息层级

采用符合苹果式排版的本地字体栈，不加载外部字体：

```css
font-family: "SF Pro Display", "SF Pro Text", "PingFang SC",
  "Segoe UI Variable", "Microsoft YaHei", sans-serif;
```

- 页面标题使用更大的字号、紧凑字距和较高字重建立产品感。
- 正文与密集数据保持清晰、克制，不通过颜色制造层级。
- 数字指标和技术字段可以继续使用等宽字体，但颜色仍遵循主题令牌。
- 仅在登录页和页面级标题区域扩大留白；高密度工作区继续采用 8px 基础间距系统，避免牺牲效率。

## CSS 架构

### 独立主题层

新增 `cdp-web/src/styles/gallery-white.css`，由 `main.js` 在现有 `cdp-global.css` 之后导入。该文件负责：

1. 全局主题令牌；
2. Element Plus 的主题映射；
3. 应用壳、通用按钮、输入框、分段导航、卡片、表格和浮层规则；
4. 各主要页面仍需统一的少量跨组件样式。

现有 `cdp-global.css` 中的 “Apple Clean Pro visual refresh” 暖色覆盖块应移除，不能再在新主题后叠加第三层补丁。主题层必须是唯一的最终视觉来源。

### 组件局部样式

`App.vue`、`LoginView.vue`、`NormalMode.vue`、`BatchMode.vue`、`SolutionCenter.vue`、`TaskCenter.vue`、`DynamicForm.vue`、文件夹树及相关弹窗中的硬编码橙色和暖色应改为语义令牌。

组件可以保留现有结构和 scoped CSS，但不得重新定义全局颜色体系。新增 class 只能用于表达呈现状态，不得改变业务状态或事件流。

### Element Plus

- 通过 CSS 变量和稳定的项目 class 覆盖按钮、输入、选择器、单选组、表格、抽屉、对话框、标签和消息。
- 主要按钮为黑底白字；危险按钮只在危险操作场景使用红色。
- 输入焦点使用橙色边界与低透明度焦点环。
- 禁用状态使用冷灰底、灰色文字和中性边界，不使用暖色或降低到不可读透明度。
- 选中态优先使用黑色文字、橙色细边或小型指示器，避免整块浅橙背景。

## 页面设计

### 登录页

- 使用纯白产品叙事区和 `#f5f5f7` 登录区形成两段式构图。
- 标题以黑色大字号和充足留白建立产品感，不使用背景光斑、渐变网格或暖色纸张效果。
- 表单按钮为黑色；输入焦点使用橙色；错误信息使用红色文字和图标。

### 全局应用壳

- 顶栏使用白色或高透明冷白，不使用暖色玻璃效果。
- 主导航使用浅灰胶囊容器与黑色选中项。
- 用户头像为黑底白字；退出悬停可以使用橙色文字，但不增加橙色背景。
- 后端离线提示改为白色容器、红色图标/窄边和深色正文，避免整条高饱和红色横幅。

### 工作台

- 保留当前三栏效率布局及现有模式切换。
- 左右侧栏使用 `#f5f5f7` 或白色，中间画布为纯白；分区只依赖中性分隔线。
- 主要计算、生成或保存行动使用黑色按钮。
- 当前字段、节点、卡片或输入焦点使用橙色细边/指示器；普通卡片不带橙色阴影。
- 批量模式、动态表单、AI 辅助抽屉和 JSON 预览遵循同一套令牌。

### 方案中心

- 方案列表、编辑器和设置栏保持白底或冷灰分区。
- 当前方案使用橙色细边或左侧指示线，不使用浅橙渐变背景。
- 草稿、已发布和归档状态主要使用文字和形状区分；只有真正的业务状态才使用 P1 语义色。
- 骨架屏使用中性灰阶动画，不出现橙色 shimmer。

### 任务中台

- 高密度指标、表格、标签和历史记录回归灰阶，减少竞争性高亮。
- 当前执行阶段使用橙色圆环或小型指示器。
- 成功、等待、失败分别使用绿、黄、红圆点/图标与明确文字，不能只靠颜色传达状态。
- 数据表格使用中性表头和分隔线；橙色只用于重点数值或当前选择。

## 数据流与行为边界

本次改造不改变 Vue 响应式状态、组件 props/emits、路由/模式切换、API 请求、认证检查、任务轮询、表单校验或本地持久化。

数据流保持：现有业务状态生成现有 class 或 Element Plus 状态；主题层根据这些状态渲染新的视觉。若某个状态缺少稳定 class，只允许添加纯呈现 class，不创建重复业务状态。

## 错误、加载、空状态与禁用状态

- 错误：红色图标/文字/窄边，容器保持白色；阻断错误必须含可读说明和恢复行动。
- 警告：黄色圆点或图标配深色文字，不使用大面积黄色背景。
- 加载：中性灰骨架或黑灰进度；当前进度点可以使用橙色。
- 空状态：白色或冷灰背景、黑灰插图/图标和明确的下一步行动，不使用橙色光晕。
- 禁用：冷灰底、灰色文字和中性边界，满足可读性要求。

## 可访问性

- 文字和控件达到 WCAG 2.1 AA 对比度。
- 焦点状态始终可见，橙色焦点环之外还保留边界变化。
- 成功、等待、失败和选中状态同时提供文字、图标或形状，不只依赖颜色。
- 键盘导航顺序和现有交互语义保持不变。
- 在 `prefers-reduced-motion: reduce` 下关闭非必要位移和动画。

## 测试策略

### 自动化主题测试

将现有 `cdp-global.apple-clean.test.mjs` 调整为 Gallery White 主题测试，至少验证：

- 存在完整的中性色、橙色和 P1 状态色令牌；
- 主要按钮为黑色，输入焦点为橙色；
- 主题中不存在暖白 `#fffdf8`、棕色文字 `#211813`、橙色阴影和橙色装饰渐变；
- 普通卡片和面板不使用彩色背景；
- 禁用状态保持可读；
- `prefers-reduced-motion` 规则继续存在。

### 组件与构建回归

- 运行现有 Vue 组件测试，确保工作台、方案中心和任务中台行为未改变。
- 运行 Vue 类型检查和 Vite 生产构建。
- 对修改过 scoped 样式的组件补充必要的结构或主题断言，避免重新出现硬编码暖色。

### 视觉检查

在至少 1920×1080、1440×900 和 1366×768 三个桌面视口检查：

- 登录、认证检查和登录错误；
- 工作台默认、选中、焦点、加载、空状态和弹窗；
- 方案中心列表、编辑、草稿、发布、空状态和骨架屏；
- 任务中台运行、成功、等待、失败、禁用和历史记录；
- 抽屉、对话框、表格、下拉层和消息提示。

## 验收标准

- 所有主要页面的可见背景和表面只使用纯白、冷灰或必要的中性透明度。
- 主题中不存在暖黄/米白画布、棕色主文字、橙色装饰渐变或橙色阴影。
- 正常页面视口内橙色只出现在少量焦点、选中、链接或重点数据上，不形成大面积色块。
- 主要行动按钮为黑底白字，危险行动和业务状态使用正确的 P1 语义色。
- 四个主要页面及其可达子界面具有一致的按钮、输入、卡片、导航、表格、状态和动效规则。
- 业务流程、API、认证、任务执行、方案编辑和数据持久化行为保持不变。
- 现有回归测试、主题测试、类型检查和生产构建全部通过。

## 风险与控制

- **CSS 优先级冲突：** 删除旧暖色主题块，避免继续堆叠 `!important`；新主题作为唯一最后导入层。
- **局部硬编码遗漏：** 对已知橙色和渐变做全仓扫描，并以主题测试阻止回归。
- **高密度页面过度留白：** 页面级标题和登录页扩大留白，工作台与任务中台继续使用紧凑 8px 间距体系。
- **状态辨识下降：** P1 状态色必须配合文字、图标或形状，并验证对比度和键盘焦点。
- **“苹果风”变成通用模板：** 保留 CDP 的三栏工作流、数据节点和任务阶段作为产品识别，不复制消费类官网的低信息密度布局。
