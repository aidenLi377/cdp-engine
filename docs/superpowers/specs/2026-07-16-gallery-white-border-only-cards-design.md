# Gallery White C：全白边框化

## 背景

A「纯白侧轨」已经消除了三栏布局左右两侧的大面积冷灰，但只处理了全高侧轨。用户查看实机后确认切换到 C「全白边框化」：三大已认证页面中的持久卡片和内层容器也使用纯白，视觉层级只依靠发丝边框、留白和排版，不再依靠半透明白、冷灰底或悬浮阴影。

## 已确认方向

- 工作台、方案中心、任务中台的页面画布、全高侧轨、持久卡片和内层信息容器统一使用 `--ui-surface: #ffffff`。
- 卡片边界使用 `--ui-divider: #e8e8ed`；需要交互反馈时，只将边界提升为 `--ui-control-border: #d2d2d7`。
- 卡片默认态、悬停态、选中态不使用灰色填充、彩色填充、投影或位移制造层次。
- Signal Orange 继续只用于小面积焦点、计数、状态点和必要提示；P1 成功、警告、危险语义色保持不变。

## 表面范围

### 工作台

- `.workbench-section`、`.panel-name-area`、`.json-code`、`.summary-compute`、`.final-list-area` 使用纯白与中性边框。
- `.left-panel-edge-toggle` 使用纯白、发丝边框和无阴影；方案态只允许橙色文字或边框，不使用暖色底。
- 已纳入 Gallery White 卡片组的行为卡片、方案卡片、摘要节点和 JSON 预览继续保持纯白。

### 方案中心

- `.creating-custom-field-panel` 与 `.custom-field-item` 的持久表面使用纯白和中性边框。
- 方案列表卡片、编辑器工具栏、设置面板与空状态容器继续保持纯白。
- 卡片悬停只改变边框，不增加投影、抬升或灰底。

### 任务中台

- `.tc-test-col`、`.tc-tags-card`、`.tc-progress-card`、`.tc-results-card`、`.tc-history-card` 保持纯白和中性边框。
- 可用 `.tc-tag-item` 与 `.tc-history-item` 使用纯白；悬停只提升边框，不使用冷灰填充、投影或位移。
- 禁用标签、禁用按钮、设置项悬停、输入控件和小型空状态仍可使用 `--ui-fill`，以保留状态识别与可读性。

## 明确保留的灰色

- 分段控件的未选中项。
- 输入控件、禁用态、设置项悬停和小型空状态插图。
- 这些灰色只能出现在小面积控件中，不能形成连续的大块页面底色或卡片底色。

## 技术边界

- 只允许修改 `cdp-web/src/styles/cdp-global.css` 和 `cdp-web/src/styles/cdp-global.gallery-white.test.mjs`。
- 不修改 Vue 模板、脚本、数据流、接口、后端、依赖或构建配置。
- 不调整布局尺寸、交互行为、文案、权限或业务状态。
- 使用 Gallery White 最终主题层覆盖旧样式，避免触碰原组件逻辑。

## 验收标准

- 1440×900 下，三大页面中不存在面积明显的冷灰或半透明卡片底色；持久卡片为纯白。
- 工作台右侧名称区、左侧库容器、方案中心创建面板、任务中台标签与历史项均通过发丝边框而非填充或阴影分层。
- 悬停卡片无投影、无位移、无灰底；焦点和选中状态仍清晰。
- 小型控件灰、Signal Orange 和 P1 状态色保留且不扩大使用面积。
- Gallery White 聚焦契约、全量前端测试和生产构建全部通过。
- 浏览器复核工作台、方案中心和任务中台，无新增控制台错误或错误遮罩。

## 回退

若 C 过于扁平，只回退本补充中的内层卡片纯白化与无阴影规则，保留已确认的 A 纯白侧轨；不回退整体 Gallery White 黑白橙与 P1 语义色体系。
