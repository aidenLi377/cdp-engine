# Gallery White 控件统一与参数精确联动设计

## 背景

Gallery White C 已将页面大表面改为纯白与发丝边框。用户在实机中继续标注了 12 个问题：导航、分段按钮、JSON 切换和下拉选项仍存在灰底或形状不一致；方案使用模式的参数选择会高亮整个行为卡，却没有同步高亮右侧摘要；同时需要精简标题并调整任务按钮文案。

本轮采用已确认的 A「独立白色胶囊」方向。目标是彻底移除这些控件的灰色环境填充，以统一几何和黑白反差建立高级感；参数联动只作用于准确字段，不扩大到整个行为节点。

## 已确认交互

- 点击方案使用区顶部参数卡，选中状态持续存在。
- 同一参数的顶部卡片、中栏对应具体字段、右侧摘要对应行同步高亮。
- 中栏整个行为卡不高亮，不出现整卡橙色边框。
- 再次点击同一参数取消高亮。
- 收缩功能继续独立工作，不再承担摘要高亮的前置条件。

## 控件视觉系统

### 顶部主导航

- `.app-shell-nav .el-radio-group` 使用透明背景、无外框、4px 间距。
- 三个 `.el-radio-button__inner` 均为 30px 高、至少 84px 宽、999px 胶囊圆角。
- 默认态：`--ui-surface` 白底、`--ui-divider` 1px 边框、深灰文字、无阴影。
- 悬停态：仍为白底，只提升为 `--ui-control-border` 和深色文字。
- 选中态：`--ui-ink` 黑底、白字、同样尺寸与圆角，无额外光晕。
- 键盘焦点保留细橙色焦点环。

### 方案中心分段控件

- `.solution-library-switch` 与 `.solution-filter-group` 外层透明、无边框、无灰底、无投影。
- 每个按钮独立成 28px 高白色胶囊，边框为 `--ui-divider`，间距 4px。
- 选中项使用黑底白字；未选中和悬停均不得使用 `--ui-fill`。
- “我的方案 / 公共方案”继续等分宽度；状态过滤按钮按内容紧凑排布。

### 摘要工具栏

- `.json-tabs` 外层透明、无边框、无灰底、无内边距。
- `.json-tab`、复制按钮、去圈人主按钮和下拉触发区统一为 32px 高、8px 圆角、12px 字号。
- “摘要 / JSON”默认白底细边，选中项黑底白字。
- 复制继续是黑色主要操作；去圈人保持白色中性操作，拆分箭头与主按钮同高。
- 工具栏继续单行布局，不增加栏宽，不改变已有行为。

### 全局下拉控件

- 本规则不是类目选择器的局部修补，而是覆盖全站所有菜单式下拉控件。
- 覆盖 Element Plus Select、Select V2、Tree Select、Dropdown、Cascader、Autocomplete，以及它们复用的 popper、菜单和选项结构。
- 覆盖范围包括：触发器/输入框、已选标签、下拉面板、默认选项、hover / `is-hovering`、selected、disabled、空状态和加载状态。
- 日期或时间选择器的日历面板不属于“菜单式下拉”，但其输入触发器若复用上述选择器表面规则，同样必须保持白底。
- 触发器和输入表面使用纯白；已选标签使用白底与 `--ui-divider` 细边；面板使用白底、中性细边与克制阴影。
- 默认选项保持白底。悬停反馈使用橙色文字或 2px 内嵌左侧橙色细轨，不使用灰色、橙色块或额外阴影。
- 已选择项仍为白底，通过深色文字、橙色细信号或勾选图标表达状态，不使用灰底。
- 禁用项仍为白底，通过三级文字色、降低不透明度和禁用光标表达状态，不使用 `--ui-fill` 或其他灰色背景。
- 空状态和加载状态所在的下拉面板保持白底，仅用文字层级和轻量图标表达语义。
- 任何下拉相关的标签、菜单项及其伪类状态都不得以灰色背景作为反馈；该规则优先于现有通用禁用态填充规则。

## 参数精确联动

### 单一状态源

继续使用 `NormalMode.vue` 的 `highlightedCfId`：

- `null`：无参数高亮。
- 自定义字段 ID：该参数被选中。
- `onHighlightCf(cfId)` 保持现有切换行为；点击相同 ID 清空，点击不同 ID 切换。

### 三处消费者

1. 顶部参数卡继续通过 `cf-use-card-active` 表示选中。
2. `DynamicForm` 继续通过现有 `isFieldHighlighted(nodeId, fieldKey)` 只高亮绑定字段。
3. 右侧摘要行改为依据 `highlightedCfId` 和 `(nodeId, fieldKey)` 精确匹配，使用 `summary-row-highlighted`。

### 移除整节点高亮

- 仅从 `workbenchMode === 'solution-use'` 分支的 `.node-wrapper` 移除 `node-highlighted` 绑定。
- 不删除 `isNodeHighlightedForCf` 或自由搭建分支的现有兼容代码，避免无关重构。
- `collapsedCfId` 只控制收缩、展开和节点筛选；摘要高亮不再依赖它。

### 数据与错误边界

- 本轮不创建新持久状态，不修改方案数据结构，不发送新请求。
- 参数 ID 不存在或绑定字段不存在时，仅不显示高亮，不抛错、不改变当前方案。
- 参数值修改、保存、撤销、重做和任务执行流程保持原样。

## 文件与技术边界

允许修改：

- `cdp-web/src/App.vue`：仅可见标题。
- `cdp-web/src/components/NormalMode.vue`：精确参数联动模板与最小状态读取。
- `cdp-web/src/components/TaskCenter.vue`：仅用户可见文案。
- `cdp-web/src/styles/cdp-global.css`：胶囊、工具栏、选择器和精确高亮样式。
- 对应前端契约测试：`App.navigation.test.mjs`、`NormalMode.customFields.test.mjs`、`TaskCenter.dmpParity.test.mjs`、`cdp-global.gallery-white.test.mjs`。

禁止修改：

- 后端、接口、扩展消息协议、依赖、构建配置、路由和权限。
- 方案序列化、任务执行函数、数据请求和持久化逻辑。
- 登录页品牌脚注、页面布局宽度和三栏结构。

## 测试策略

### TDD 契约

- App 契约：可见标题包含“圈选工作台”且不含“CDP 圈选工作台”。
- 参数联动契约：方案使用分支不再绑定 `node-highlighted`；摘要行由 `highlightedCfId` 驱动；匹配函数读取 `highlightedCfId`；现有字段级高亮保留。
- 任务契约：两个空闲按钮均为“运行”，旧“测试”按钮文案不存在，空状态提示同步。

## 回退

- 控件样式可独立回退到当前 C 表面规则，不影响参数状态。

- 文案变化可独立回退，不影响函数和接口。
