# DataBank 人工应用确认门设计

## 目标

保留 DataBank 最终“应用/确定”按钮由人工点击的安全机制，同时保证网页不会在人工操作前误报“推送已发起”或提前查询 DataHub。

## 设计

数据引擎推送拆成三个明确阶段：

1. 扩展搜索人群、选择阿里妈妈和达摩盘，并打开最终确认弹窗；扩展不点击最终按钮。
2. DataBank 标签页保持前台，网页显示“等待人工确认”。网页通过独立消息请求扩展等待；扩展观察弹窗，只有相关“应用/确定”控件连续消失 1.5 秒后才判定人工步骤结束。
3. 人工步骤结束后才打开 DataHub 查询本次人群状态；只有状态为“已应用”才显示推送完成。

## 消息接口

- 网页到扩展：`CDP_AUTOMATE_DATABANK_WAIT_APPLY`
- 后台到 DataBank 内容脚本：`AUTOMATE_DATABANK_WAIT_APPLY`
- 成功响应：`{ ok: true, step: 'manual_apply_confirmed' }`
- 最长等待时间：30 分钟；超时返回清晰错误，不进入 DataHub。

## 边界与安全

- 扩展绝不调用最终按钮的 `click()`。
- 弹窗出现后不切回网页，避免用户找不到确认窗口。
- 若用户取消弹窗，仍会进入 DataHub 核验；DataHub 未出现“已应用”时任务不会成功。
- 不修改 DataBank 参数粘贴、DMP 透视、结果计算、字段显示、Rebase 或账户方案隔离。

## 验证

- 测试确认打开弹窗后没有点击最终按钮。
- 测试确认弹窗未关闭时等待消息不返回成功。
- 测试确认人工关闭弹窗后才返回 `manual_apply_confirmed`。
- 测试确认网页收到人工确认后才调用 DataHub。
- 运行前端、内置扩展、合并扩展和后端全量测试。
