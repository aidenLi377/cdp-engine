
# 🚀 CDP 客户数据平台 - 配置驱动的人群圈选引擎 (CDP-Engine)

基于 **Vue 3 + Element Plus** (前端) 和 **Python**（后端）构建的轻量级、高拓展性的人群圈选 JSON DSL 生成引擎。

本项目彻底摒弃了传统的“前端硬编码表单 + 后端硬编码拼接”的低效模式，采用**纯配置驱动 (Config-Driven)** 的架构思想：所有复杂的联动逻辑、下拉菜单、模板状态机均由底层的 CSV 维表动态接管。

## ✨ 核心特性

**：前端零硬编码，所有 UI 组件（单选、多选、区间、日期组合）和后端的映射逻辑，均通过`1.参数表.csv`等多张维表自动生成。
- **⚡️ 海量数据极速渲染**：针对超大规模维表（如 1.6 万+ 品牌数据），深度集成 Element Plus 的 `el-select-v2` 虚拟滚动技术，保证页面丝滑不卡顿。
-**智能状态机路由**：对于金额、频次等“开闭区间”逻辑（如大于10元、200-500元），引擎能自动识别`min_only、range等状态，并精准匹配生成OPEN_CLOSE/CLOSE_CLOSE等底层标准 DSL 报文。
- **📅 复杂时间模型接管**：支持“相对时间（如最近30天）”与“绝对时间（如日历区间）”的双轨解析，并自动转换为官方底层所需的结构化参数化 JSON。
- **🛡️ 幽灵参数清洗**：内置强大的脏数据拦截机制，前端组件在 `v-if` 隐藏时产生的“幽灵变量”绝不会污染最终发给后端的 JSON 报文。

## 📂 项目结构

```text
CDP_Project/

## 🚀 快速启动

本项目为前后端分离架构，需要分别启动数据引擎（后端）和交互大堂（前端）。

### 1. 启动底层数据引擎 (Python 后端)

确保已安装 Python 环境及所需的依赖（如 Flask 等）。
在根目录下运行：

```bash
python app.py

```

> 后端服务默认运行在 `http://127.0.0.1:5000`

### 2. 启动交互大堂 (Vue 前端)

进入前端子工程目录：

```bash
cd cdp-web

```

安装所有前端依赖（首次运行需要）：

```bash
npm 安装

```

启动本地开发服务器：

```bash
npm 运行开发

```

> 前端服务默认运行在 `http://localhost:5173` (或 5174)，点击终端输出的链接即可在浏览器中访问。

## 💡 设计哲学

**“如果业务逻辑发生改变，能在维表上修改的，就绝不在代码里修改。”**
代码只提供健壮的框架和容错机制，将控制权完全交还给数据运营与策略架构师。
