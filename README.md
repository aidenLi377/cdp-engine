# 🎯高效人群圈选引擎 (CDP Condition Configuration Engine)

## 📖 项目简介

本项目是一个为 CDP（客户数据平台）量身打造的**动态条件配置引擎**。

有别于传统的硬编码表单，本系统采用**“后端 Schema 驱动 + 前端动态渲染”**的架构。通过动态解析后端的参数表与逻辑维表，前端能够自动化构建出包含复杂联动、严密业务边界与极致防呆设计的配置界面，极大提升了业务人员在进行“人群圈选”时的效率与准确率。

## 🚀 核心特性

### 1. 🧠 数据驱动的动态渲染 (Schema-Driven UI)

- **动态解析**：后端基于 `Pandas` 解析 CSV 配置表与逻辑矩阵，实时生成前端所需的渲染 JSON。
- **高扩展性**：新增表单字段或行为包（如“AIPL状态”、“商品行为”、“类目公域行为”）无需修改前端代码，只需更新配置表即可热生效。

### 2. ⏱️ 智能时间区间引擎 (Smart Time Controller)

- **动态底线推算**：重构了复合时间组件，支持“过去 N 天”与“固定日期”无缝单选。
- **业务规则隔离**：深入包级别的独立校验。例如在【类目公域行为】中，纯选“购买”行为可追溯 **2 年** 数据；一旦混选其他行为（或处于其他包中），系统触发**“短板效应”**，强制将可选范围降级并物理锁定在 **366 天**，确保底层数据查询的绝对安全。
- **智能数据类型通吃**：底层拦截器完美兼容原生 `Date` 对象与 `YYYYMMDD` 字符串，杜绝日期组件崩溃。

### 3. 🛡️ 专业级防呆机制 (Ultimate Anti-Fool Logic)

- **动态物理拦截**：基于不同行为包动态设置输入上限。例如“商品标题关键词”在【类目公域行为】中物理锁死最多输入 10 个，在【商品行为】中自动放宽至 50 个。
- **独占互斥清空**：针对多选下拉框的“全部”选项实施霸道互斥锁。新选“全部”自动清场；已有“全部”时选其他项则自动踢出“全部”，从根源切断脏数据。
- **类型自动适配**：智能识别单选/多选组件需求，自动在 `String` 与 `Array` 之间转换初始化数据。

### 4. 💅 极致的 B 端交互体验 (Polished UX)

- **呼吸感布局**：摒弃传统表单左侧拥挤的文字说明，将动态提示语平移至输入框右侧，左侧仅保留清爽的悬浮 `ⓘ` 提示。
- **动态蓝标计数器**：为多选和列表输入组件挂载实时跳动的蓝色计数标签（如 `已选 13/50`），赋予用户极强的掌控感。
- **无残影搜索**：关闭多选折叠 (`collapse-tags`) 让选项直观平铺；引入即时清空机制 (`reserve-keyword="false"`)，选中目标后瞬间清除搜索残影，保障连续输入的丝滑体验。

## 🛠️ 技术栈

- **前端 (Frontend)**: Vue 3 (Composition API) + Element Plus
- **后端 (Backend)**: Python 3 + Flask + Pandas
- **架构设计**: RESTful API + JSON Schema 动态表单驱动
